# Lab3 — CBLAS Level 3: DGEMM

Реализация операции третьего уровня BLAS (DGEMM) на C++ с параллелизацией через `pthread`.
Выполнено сравнение производительности с OpenBLAS.

---

## Операция

| Операция | Тип    | Уравнение       | Сложность |
| -------- | ------ | --------------- | --------- |
| DGEMM    | double | C = α·A·B + β·C | O(n³)     |

---

## Алгоритм

Используется блочное умножение матриц (**cache blocking**) с дополнительными оптимизациями:

* матрицы A и B разбиваются на блоки (tiles)
* данные упаковываются в непрерывные буферы (Ap, Bt)
* B транспонируется для последовательного доступа
* используется **microkernel 4×4** (вычисление блока C за один проход по K)
* строки матрицы C распределяются между потоками (`pthread`)
* компиляция с `-O3` → автовекторизация (NEON на Apple M1)

---

## Параметры блокирования

```
MC = 64   (строки блока A)
KC = 512  (общее измерение)
NC = 64   (столбцы блока B)
```

Размер блока подобран с учётом кешей Apple M1 (L1/L2).

---

## Параллелизм

* используется `pthread`
* матрица C делится по строкам между потоками
* тестирование проводится для 1 / 2 / 4 потоков

---

Каждый тест выполняется **10 раз**, считается геометрическое среднее.

---

## Результаты (Apple M1, N=512)

```
=== DGEMM (double)  N=512 ===
OpenBLAS  geomean = 0.0033 s

threads  run    my (s)       ob (s)       rel %%
------------------------------------------------
...
>> threads=1   geomean = 27.0% от OpenBLAS
>> threads=2   geomean = 49.5% от OpenBLAS
>> threads=4   geomean = 81.7% от OpenBLAS
```

---

## Вывод

Реализована оптимизированная версия DGEMM с использованием:

* cache blocking
* упаковки данных (packing)
* microkernel 4×4
* многопоточности (pthread)

Полученная реализация достигает **~82% производительности OpenBLAS**,
что удовлетворяет требованию лабораторной работы (не хуже 70%).

OpenBLAS остаётся быстрее за счёт:

* более сложного register blocking
* ручной оптимизации под архитектуру (assembly)
* более продвинутого планирования потоков
