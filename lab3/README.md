# Lab3 - BLAS Level 3: DGEMM + TRSM

Реализация операций третьего уровня BLAS на C++ с параллелизацией через `pthread`.
Выполнено сравнение производительности с OpenBLAS.

---

## Реализованные операции

| Операция | Тип    | Уравнение       | Сложность |
| -------- | ------ | --------------- | --------- |
| DGEMM    | double | C = α·A·B + β·C | O(n³)     |
| TRSM     | double | B = α·A⁻¹·B     | O(n³)     |

---

## Алгоритмы

### DGEMM (основная реализация)

Используется оптимизированное блочное умножение:

* cache blocking (MC, KC, NC)
* упаковка данных (packing A и B)
* транспонирование B для последовательного доступа
* microkernel 4×4
* параллелизм по строкам (pthread)
* автовекторизация (NEON, `-O3`)

Основной упор сделан на DGEMM как базовую операцию BLAS Level 3.

---

### TRSM (triangular solve)

Реализован базовый алгоритм **forward substitution**:

* нижнетреугольная матрица (Lower)
* левое умножение (Left)
* без транспонирования (NoTrans)
* ненормированная диагональ (NonUnit)

Алгоритм:


for i = 0..n-1:
    B[i] = (B[i] - Σ A[i,k]·B[k]) / A[i,i]


---

## Особенности TRSM

В отличие от DGEMM:

* присутствуют **зависимости между строками**
* невозможно эффективно распараллелить по строкам
* ограничена векторизация
* хуже использование кеша

Поэтому TRSM значительно уступает GEMM по производительности.

---

## Параметры блокирования (DGEMM)


MC = 64
KC = 512
NC = 64


---

## Параллелизм

* используется `pthread`
* DGEMM масштабируется по потокам
* TRSM масштабируется ограниченно из-за зависимостей

## Вывод

* Реализованы операции BLAS Level 3: DGEMM и TRSM
* DGEMM оптимизирован с использованием cache blocking и microkernel
* достигнуто ~82% производительности OpenBLAS

TRSM реализован корректно, однако:

* имеет последовательные зависимости
* хуже распараллеливается
* значительно уступает OpenBLAS

OpenBLAS превосходит реализацию за счёт:

* register blocking
* SIMD (NEON)
* ручных оптимизаций (assembly)
* продвинутого планирования потоков
